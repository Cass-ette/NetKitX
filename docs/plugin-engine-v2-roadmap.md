# Plugin Engine 2.0 Roadmap

## 愿景

让 NetKitX 插件系统从"工具集合"进化为"安全工具引擎",支持从简单脚本到复杂应用的全场景。

---

## 四个层级

```
Level 0: 单次执行 (现有)
  ↓ 简单,5 分钟上手
Level 1: 会话模式 (多次交互)
  ↓ 保持状态,连续操作
Level 2: UI 模式 (自定义界面)
  ↓ 可视化,用户体验
Level 3: 服务模式 (长期运行)
  ↓ 守护进程,持续监控
```

---

## Phase 1: 会话模式 (MVP)

**时间**: 2-3 天
**优先级**: P0 (必须)

### 核心功能
- [x] 会话生命周期管理
- [x] WebSocket 双向通信
- [x] Redis 会话持久化
- [x] 会话恢复机制

### 交付物
- WebShell 插件 2.0 (会话模式)
- 会话管理 API
- 前端会话 UI

### 验收标准
- WebShell 可以连续执行命令
- 刷新页面会话不丢失
- 支持 50 个并发会话

## 风险和缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| WebSocket 不稳定 | 高 | 中 | 自动重连 + 消息队列 |
| 性能瓶颈 | 中 | 高 | 压测 + 优化 |
| 用户学习成本 | 中 | 中 | 文档 + 示例 |
| 安全风险 | 高 | 低 | 沙箱 + 权限控制 |
